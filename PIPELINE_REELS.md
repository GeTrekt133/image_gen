# PIPELINE_REELS — перенос движения из рилсов на персону (LTX-2.3), рабочее состояние 2026-07-09

**TL;DR.** Дрифт пикселей и «сгенерированный» вид побеждены. Главные причины были: (1) **вклеенные кадры/склейки в стоковом драйвинг-видео** (обнаружено оператором), (2) одностадийная генерация на 512×896 вместо официальной двухстадийной, (3) отклонения от официального рецепта (sampler, силы LoRA, guide strength). Финал дня: **полнокадровый режим v8** — фон генерируется целиком (не вырезается маской из реального видео), стабильность фона лучше реальной съёмки (bgNoise 2.55 vs 4.3), шва/ореола нет по построению.

Все прогоны — форки, старые файлы не тронуты. GPU: RTX 6000 Blackwell 96GB, чекпоинт `ltx-2.3-22b-dev-fp8`.

### v8 full-frame — как устроен (новый дефолт)
Идея (предложение оператора): **выкинуть частичную генерацию вообще**. Из драйвинга берётся ТОЛЬКО DWPose-скелет; все пиксели кадра, включая фон, генерируются моделью → стыка «реальное/сгенерированное» не существует, ореол невозможен.
- База: `wf_ltx_dance_notile.json` (ветка с i2v-якорем) + фиксы рецепта + unmasked stage-2.
- **Якорь фона**: `LTXVImgToVideoConditionOnly` strength **0.7** + `LTXVPreprocess(18)` (цифры из официального V2V-графа), image = кадр 0 базовой генерации v5 (персона в «той самой» комнате в позе первого кадра драйвинга). Комната стартует правильной и держится через temporal attention: bgNoise 2.55 — стабильнее реальной съёмки.
- ⚠ В notile-графе размер латента диктует **pose-картинка** → `DWPreprocessor.resolution` ОБЯЗАН равняться целевому shorter-size (768), иначе stage-1 тихо падает до 512p.
- Склейки в драйвинге не мешают (как в v5): один проход, одна персона, один лук.
- Компромисс: комната «по мотивам» (детали приблизительные, не документальные). Если нужна точная геометрия — план Б: второй гайд depth (DepthAnythingV2 стоит локально), не реализован.

(`run_v4_fullreel.py` — посегментная генерация с разными промптами целиком; идентичность между сегментами плывёт, использовать только если нужны разные сцены/свет, а не только одежда.)

**Ключевые открытия:**
- **Single-pass через склейки работает**: LTX-2.3 отрабатывает жёсткий кат в драйвинге как монтажный кат (без ghosting) и держит **одну персону и один аутфит на весь клип** — общий temporal attention. 241 кадр (31 латентный) на 1152×2016 проходит без OOM, 441s.
- **Outfit-swap**: драйвинг = собственный выход v5, маска = силуэт rembg **минус эллипс лица** → лицо остаётся буквальными пикселями v5 (идентичность залочена физически), одежда перегенерируется под промпт сегмента. Сегменты не 8n+1 → паддить замороженным последним кадром до ближайшего 8k+1, паддинг отбросить при сборке. ~65s/сегмент.

## Чеклист подготовки драйвинга (`prep_drive_new.py`)

1. **Детектор склеек — обязательно для каждого рила**: межкадровый |Δ| (grayscale, 180×320), склейка = спайк > медиана + 8·MAD. Стоковые рилсы часто монтажные (DZLY571hMqd: каты на 134/171/208; DZPPRW: скрытая вклейка — именно она давала «необъяснимый» дрифт).
2. Выбрать окно **8n+1 кадров** без склеек (или сегменты для v4/v5), отступая 2–3 кадра от каждого ката (transition-кадры полусмешаны).
3. **Нативный fps** — никакого ресемпла 30→24 (дубли/выпадения = периодический дёрг). LTX нормально принимает 30 fps через `GetVideoComponents` → `LTXVConditioning`.
4. 720×1280 → scale 512×910 lanczos → центр-кроп 512×896.
5. **Вшитые сабтайтлы** (почти во всех рилсах): детект полосы устойчиво-белых строк (V>225, S<40, >25 пикс/строку в среднем по окну между катами) → LaMa (`simple_lama_inpainting`) по полосе на всех кадрах окна. Порог может пропустить короткую надпись — обычно её съедает masked-regen, но проверять глазами.
6. Маска: rembg `u2net_human_seg`, alpha>30, dilate 21×21 ×2, GaussianBlur 9 → h264 crf 6.

## Рецепт генерации (граф `wf_vace_v2.json` + hires-патчи в раннерах)

Двухстадийный masked-regen с реальным фоном (VACE-аналог на открытой union IC-LoRA):

**Stage 1 (768×1344):** драйвинг ×1.5 → `VAEEncodeTiled` (база = реальное видео) → маска → `LTXVDilateVideoMask`(16) → `LTXVPreprocessMasks`(grow 8, **clamp_min 0.0**) → `LTXVSetVideoLatentNoiseMasks` → латент в `LTXAddVideoICLoRAGuide` (**strength 1.0**, pose-гайд = DWPose тех же ×1.5 кадров — единая resize-цепочка!) → distilled-LoRA **0.5** + union IC-LoRA **1.0**, 8-step ManualSigmas, **euler_ancestral_cfg_pp**, cfg 1, негатив официальный (`pc game, console game, ...`).
→ `LTXVLaplacianPyramidBlend` с реальными кадрами (фон = 100% реальные пиксели).

**Stage 2 (1152×2016):** бленд ×1.5 → `VAEEncodeTiled`(512/64/500/64) → те же noise-маски ×1.5 → хвост `0.725, 0.4219, 0.0`, **euler_cfg_pp**, cfg 1 → decode 2×2 → финальный Laplacian-бленд с реальными кадрами ×1.5.

**Промпт:** только внешность/свет/материалы, **никаких motion-слов** («dancing» и т.п. — конфликт с IC-LoRA, даёт пластик и нестабильность). Одежда обязана вписываться в силуэт маски. Seed фиксированный (42).

Тайминги: 121 кадр ≈ 200s; 241 кадр ≈ 440s; outfit-swap сегмент 41f ≈ 65s.

## Проверено и отвергнуто

- `detect_face=disable` в DWPose — лицо плывёт сильнее (точки нужны).
- Savitzky–Golay сглаживание кейпоинтов (`run_v2e_smoothpose.py`) — скелет стабильнее, выход не лучше.
- «Pro» stage-1 (distill 0, 30 steps, cfg 3) — самое фотографичное лицо, но дороже (~260s) и метрика дрифта хуже; v2c выбран оператором.
- Пост-композит матированной персоны — выглядит «наклеенным» (см. память `ltx-bg-from-video`).

## Ореол по границе маски (masked-режимы) — история борьбы, итог: НЕ ЛЕЧИТСЯ, уходить в v8

Ген-зона отличается от реального фона (экспозиция от промпта + fp8-микротекстура) — Laplacian-бленд прячет шов, но на плоской стене виден «контур маски» шириной ~150px (dilate 32 + `mask_low_res_dilation 5`).
Перепробовано и ОТВЕРГНУТО (не повторять):
1. `fix_v6_color.py` — глобальный аффинный цветофит по кольцам сырой маски: no-op (кольца попадали на волосы/тело, рассинхрон локальный, глобальный gain не может его чинить).
2. `fix_v6_color2.py` — локальное поле (inpaint-эталон стены + masked blur): рисует тёмную «стикерную» окантовку вплотную к персоне.
3. `run_v7_fullchain.py` — фикс в графе: ColorMatch(mkl) перед блендами + узкая растушёвка + промпт без света. Ореол остался — различие не только в экспозиции, стык реального и сгенерированного на плоской стене виден всегда.
Вывод: гибрид «реальные пиксели + генерация» на однотонных поверхностях не сшивается. Решение — полнокадровый v8.

## Открытые хвосты (в порядке ценности)

1. **char-LoRA персоны** (LTX Trainer) — единственное, что даст лицо именно нейродевушки во всех генерациях. Сейчас идентичность прибивается только v5/v6-механикой внутри клипа.
2. **Ingredients IC-LoRA** (`ltx-2.3-22b-ic-lora-ingredients-0.9`, официальный воркфлоу есть в example_workflows) — генерация по референс-листу персонажа; проверить гейт на HF и совместимость с pose-контролем (занимает тот же guide-слот).
3. Эллипс защиты лица в v6 → пофреймовые face-landmarks из DWPose (при запрокинутой голове эллипс открывает губы → лёгкий сдвиг макияжа, сегменты 3–4 v6).
4. A/B fp8 → bf16 чекпоинт + официальный `comfy_gemma_3_12B_it` (текущий Gemma — сторонний fp8-репак; влияет на мелкую текстуру).
5. `onnxruntime-gpu` для DWPose (сейчас CPU/OpenCV — медленно).

## Метрики (нормализация кадров к 512×896)

- tNoise = средний межкадровый |Δ| по gray; bgNoise — то же по фоновой полосе; faceDrift — |Δ| по кропу головы, трекаемому из маски драйвинга; sharp = дисперсия лапласиана.
- Референсы: реальная съёмка tNoise ≈ 7–8.3; старый одностадийный пайплайн ≈ 11.2; v3 ≈ 5.1.
